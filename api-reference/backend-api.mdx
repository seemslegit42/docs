# Backend API Reference

This page documents the Nexus backend mounted under `/api/v1`.

## Base URLs

| Surface | URL |
| :-- | :-- |
| Nexus backend | `http://localhost:8001/api/v1` |
| App root | `http://localhost:8001/` |
| Health | `http://localhost:8001/health` |
| API health | `http://localhost:8001/api/v1/health` |

## Authentication

The backend uses a mix of auth gates:

| Auth method | Meaning |
| :-- | :-- |
| `check_auth` | JWT or session-gated current-user access |
| `get_current_user` | Current authenticated user is required |
| `get_user_organisation` | Authenticated user plus organisation scope |
| `validate_api_key` | API key authentication for public agent endpoints |
| `_require_task_delegator` | Custom task-delegator gate using `X-Nexus-Delegator-Role` with allowed values `architect` and `forgemaster` |
| none | Public or server-side admin-plane route |

Unless otherwise noted, request and response bodies are JSON.

## Common Errors

| Status | Meaning |
| :-- | :-- |
| `400` | Validation error, missing configuration, unsupported request, or failed external dependency |
| `401` | Authentication missing or invalid |
| `403` | Authenticated but not authorized, often organisation or role mismatch |
| `404` | Resource not found |
| `409` | Conflict, usually when a state transition is invalid |
| `422` | FastAPI request validation failure |
| `500` | Unhandled server-side error |
| `502` | Downstream sync or command execution failure |
| `503` | Service unavailable, used in a few diagnostic endpoints |

## Notes on Response Shapes

- When a controller declares a `response_model`, that model is the contract of record.
- When no `response_model` is present, the endpoint usually returns an ORM object, a helper dict, or a list of plain dicts.
- Several Appwrite-backed routes return server-side admin payloads directly instead of a normalized application schema.

## Source of Truth

The backend route registry lives in `nexus/router_registry.py`. The router inventory in this page is derived from the mounted controller files and the app-level routes in `nexus/app_factory.py`.
