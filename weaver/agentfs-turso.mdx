## What AgentFS is

[AgentFS](https://docs.turso.tech/agentfs/introduction) is **a filesystem for AI agents, built on Turso**.

- **Copy-on-write isolation** — agents run in sandboxes; source tree stays safe
- **Single-file storage** — one portable SQLite/libSQL database per session
- **Built-in auditing** — every file operation is recorded and queryable
- **Cloud sync** — optional sync to Turso Cloud ([sync guide](https://docs.turso.tech/agentfs/guides/sync))

```bash
curl -fsSL https://agentfs.ai/install | bash
agentfs run bash   # isolated shell in your project
```

SDKs: [TypeScript](https://docs.turso.tech/agentfs/sdk/typescript), [Python](https://docs.turso.tech/agentfs/sdk/python), [Rust](https://docs.turso.tech/agentfs/sdk/rust).

## How Weaver uses Turso

| Path | Purpose |
| :---- | :---- |
| AgentFS CLI / Nexus `ignite.py`, `/api/v1/agentfs/*` | CoW sandboxes, session VFS |
| `src/db/turso.ts` | libSQL client—vectors, CDC, embeddings schema |
| `src/services/vector-store.ts`, `ai-memory.ts`, `code-indexer.ts` | RAG and agent memory |
| `src/components/canvas/agentfs-sandbox-bar.tsx` | Pull/push/checkpoint on canvas |
| `src/components/panels/artifact-codex.tsx` | Files + audit tabs (AgentFS) |
| `src/hooks/use-agentfs-session.ts` | Auto-init sandbox from `workflowId` |
| Nexus `/api/v1/agentfs/*` | Session init, timeline, sync |
| `nexus/engine/swarm.py` | `SovereignSession.ensure_session()` before weave |

## Frontend entry points

- **Dock → AgentFS Codex** — `panels.codex` → Artifact Codex
- **Canvas overlay** — AgentFS sandbox bar on `WorkflowCanvas`
- **Bedrock slice** — `pullSubstrateChanges` / `pushSubstrateChanges` in store

## What Turso is *not*

Turso is **not** the reflex layer for live canvas cursors—that is **DragonflyDB** via Nexus `/api/v1/collaboration`.
