Automated build, test, security scan, deploy, and rollback for all Weaver modules.

## Pipelines

| Workflow | Trigger | Purpose |
|----------|---------|---------|
| [`.github/workflows/ci.yml`](../.github/workflows/ci.yml) | PR + push `main`/`dev` | Lint, unit tests, integration tests, security, image build |
| [`.github/workflows/cd.yml`](../.github/workflows/cd.yml) | Push `main`/`dev`, manual | Build/push GHCR images, deploy, auto-rollback on failure |
| [`.github/workflows/rollback.yml`](../.github/workflows/rollback.yml) | Manual | Roll back staging or production to previous tag |

## CI stages

1. **Python lint** ŌĆö Ruff on engine, controllers, services, targeted tests
2. **Frontend** ŌĆö `bun install`, production build (typecheck gate)
3. **Unit tests** ŌĆö Nexus `engine` + `services`, frontend `src/utils/__tests__`
4. **Integration** ŌĆö `test_weaver_api.py` (in-process) + Docker Compose CI smoke
5. **Security** ŌĆö `pip-audit`, `bandit`, `bun audit`, Trivy FS scan
6. **Build images** ŌĆö frontend, Nexus API (verify Dockerfiles)

## CD & environments

| Branch | Environment | Image tag |
|--------|-------------|-----------|
| `dev` | `staging` | `staging-<sha>` |
| `main` | `production` | `production-<sha>` + `latest` on production |

Images are published to:

```text
ghcr.io/<owner>/<repo>-frontend:<tag>
ghcr.io/<owner>/<repo>-nexus-api:<tag>
ghcr.io/<owner>/<repo>-celery:<tag>
```

## Deploy secrets (GitHub)

Configure in repo **Settings ŌåÆ Secrets and variables ŌåÆ Actions**:

| Secret | Required | Description |
|--------|----------|-------------|
| `DEPLOY_HOST` | For remote deploy | SSH hostname |
| `DEPLOY_USER` | For remote deploy | SSH user |
| `DEPLOY_SSH_KEY` | For remote deploy | Private key (PEM) |

Without `DEPLOY_HOST`, CD still **builds and pushes images**; deploy is registry-only.

On the deploy host:

```bash
mkdir -p ~/weaver-deploy
# copy docker-compose.deploy.yml + scripts/ci/*.sh
export REGISTRY=ghcr.io/owner/repo
export IMAGE_TAG=production-abc1234
./deploy.sh
```

## Rollback

**Automatic:** CD `rollback-on-failure` job runs if deploy fails (re-tags `rollback` images).

**Manual:** Actions ŌåÆ **Rollback ŌĆö Weaver Platform** ŌåÆ pick environment.

Local:

```bash
./scripts/ci/rollback.sh
```

State is stored in `.deploy/state.json` (artifact in CI).

## Local CI commands

```bash
make ci-unit          # unit tests only
make ci-integration   # integration + compose smoke
make ci-security      # security-scan.sh
```
