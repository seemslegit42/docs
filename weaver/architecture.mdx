## 2. Persistence Trinity (Canonical)

ΛΞVON OS uses **three cognitive stores** (typically **self-hosted on Coolify**). **Turso is first-class**—it is not optional; it is the substrate under **AgentFS** and Weaver’s libSQL agentic plane.

| Layer | Technology | Role | Canonical data |
| :---- | :---- | :---- | :---- |
| **Global Core** | **Appwrite TablesDB** | Sovereign’s Ledger | Identity, teams, blueprints, agents, Ξ/financial records, Storage, Transactions |
| **Reflex** | **DragonflyDB** | Live reality | Canvas UI state, cursors, presence, volatile signals, hot debugger steps |
| **Sandbox Memory** | **AgentFS on Turso** | Agent working memory | CoW VFS, tool audit, vectors, CDC, `agent.db`, LangGraph checkpoints |

```text
Appwrite (durable ledger)
Dragonfly (ephemeral reflex, ms latency)
AgentFS → Turso/libSQL (per-session agent cognition + cloud sync)
```

### Turso’s role (critical)

Per [Turso AgentFS](https://docs.turso.tech/agentfs/introduction):

- **AgentFS** = filesystem for AI agents **built on Turso** (SQLite/libSQL, CoW, audit log, optional cloud sync).
- **Weaver `src/db/turso.ts`** = `@libsql/client` bedrock: vectors, embeddings, CDC, `ai-memory`, code indexer—not a “legacy adjunct.”

**Boundary rules**

- **Appwrite** — survives restarts; permissions; Transactions API; Realtime for rows/functions.
- **DragonflyDB** — TTL reflex via Nexus `/api/v1/collaboration/*` (`REDIS_URL` / `DRAGONFLY_URL`).
- **AgentFS / Turso** — never identity or global ledger; mount via `ignite.py`; Nexus `/api/v1/agentfs/*`.
- **Do not** use Turso for new sub-second canvas/presence keys—use Dragonfly. **Do** use Turso for everything AgentFS and agentic memory require.
