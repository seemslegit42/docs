Weaver Studio is a Next.js App Router client. All product UI flows through a small set of **shell components** that lazy-load panels and registries. Backend calls go through **`src/services/api.ts`** (`apiService` → `/api/v1/*` proxy).

## App routes

| Route | File | Purpose |
| :---- | :--- | :------ |
| `/` | `src/app/page.tsx` | Landing / auth gateway |
| `/canvas` | `src/app/canvas/page.tsx` | **Primary IDE** — workflow canvas, HUD, dock |
| `/dashboard` | `src/app/dashboard/page.tsx` | Metrics hub + registry overlays |
| `/observatory` | `src/app/observatory/page.tsx` | Whole-app environment checklist |
| `/login`, `/signup`, … | `src/app/*/page.tsx` | Appwrite auth rituals |

Auth pages and canvas/dashboard both mount **`SubstrateInitializer`** (identity sync + Nexus prefetch).

## Request path to Nexus

```text
Browser (apiService)
  → GET/POST /api/v1/...
  → next.config.ts rewrite
  → http://localhost:8001/api/v1/...
  → nexus/router_registry.py
```

Server-only paths (not proxied):

| Path | Handler |
| :--- | :------ |
| `/api/observatory` | `src/app/api/observatory/route.ts` — env planes + stack probes |
| `/api/vectors/*` | Turso vector helpers (Next route handlers) |

Server actions: `src/app/actions/nexus.ts`, `collaboration.ts` → `src/services/nexus-server.ts`.

## UI shell (canvas & dashboard)

```text
SidebarProvider + Sidebar          ← registries, global rituals
Header / Footer
WorkflowCanvas (React Flow)        ← /canvas only
HudLayout                          ← panel blades (palette, console, codex, …)
OverlayManager                     ← full-screen registries + modals
SovereignDock                      ← command bar + quick tools
BeepAvatar                         ← opens AssistantPanel
```

### State

Global UI state: **`src/store/`** (Zustand) — `ui-slice.ts` (`panels`, `activeRegistryTab`), `execution-slice.ts` (weave, debug), `bedrock-slice.ts` (AgentFS sync), `collaboration-slice.ts`.

On auth, **`SubstrateInitializer`** prefetches automations, marketplace, substrate functions, and AgentFS sync stats via `apiService`.

## Purged / optional modules

These files exist but are **not** mounted in the shell (stubs or shadcn primitives):

- `function-registry.tsx`, `api-service.ts` — marked `Purged`; use Settings → Filaments and `api.ts`.
- `beep-assistant.tsx` — superseded by `AssistantPanel`.
- Unused shadcn primitives (`accordion`, `chart`, …) — import when a feature needs them.
