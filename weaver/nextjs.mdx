Weaver Studio uses **Next.js App Router** (React 19, Bun). Product code lives under `src/app/` and `src/components/`.

## Routes

| Path | Role |
| :--- | :--- |
| `/canvas` | Primary IDE — React Flow canvas, HUD, Sovereign dock |
| `/dashboard` | Metrics widgets + registry overlays |
| `/observatory` | Environment planes UI |
| `/login`, `/signup`, … | Appwrite auth |

See [Frontend wiring](/weaver/frontend-wiring) for panel/registry map.

## Nexus proxy

`next.config.ts` rewrites browser calls:

```ts
{ source: '/api/v1/:path*', destination: 'http://localhost:8001/api/v1/:path*' }
```

Use **`src/services/api.ts`** — do not hardcode `:8001` in client components.

Server-side Nexus calls: **`src/services/nexus-server.ts`** and **`src/app/actions/nexus.ts`**.

## State & services

| Layer | Path |
| :---- | :--- |
| Global store | `src/store/` (Zustand slices) |
| API client | `src/services/api.ts` |
| AgentFS | `src/services/agentfs.ts` |
| Appwrite client | `src/services/appwrite.ts` |
| Realtime | `src/services/realtime.ts` |
| Turso / vectors | `src/db/turso.ts` |

## UI conventions

- **components/** — presentation (panels, registries, canvas)
- **services/** — business logic and HTTP
- **store/** — client state
- **hooks/** — React hooks (`use-agentfs-session`, `use-workflow-collaboration`)
- Avoid new code in `src/lib/` unless it is a portable library
